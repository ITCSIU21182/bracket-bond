Bracket Bond is one Solana program and a thin TxLINE client that compose into a
proof-settled prediction market. You hold a position on a team's whole knockout
run, watch it mark up and down from real odds, exit anytime — and each round is
resolved by a Merkle proof of the actual result, not a vote.

| Piece | Runs on | What it does |
| --- | --- | --- |
| `bracket_bond` program | Solana (Anchor) | market, collateral vault, buy / sell, **proof-settled** rounds, redeem |
| TxLINE client (`scripts/txline`) | Node 20+ | subscribe, auth, odds / scores, builds the `validate_stat` CPI |
| Frontend (`app`) | Browser | market UI — buy, watch marks, exit, settlement feed |

<Callout type="info">
  **Proof, not vote.** Polymarket settles when humans agree (UMA optimistic
  oracle, with a dispute window). Bracket Bond settles when a Merkle proof of the
  real result verifies on-chain via `Txoracle.validate_stat` — automatically, no
  human in the loop, no dispute window. Collateral is native SOL (play-money on
  devnet); the TxL token is used only to authorize data, never for wagering.
</Callout>

## Architecture

One program, one verifiable data feed, three moving parts.

<Mermaid chart={`
flowchart LR
  U["User / keeper"]
  FE["Frontend (app)"]
  REL["TxLINE client (scripts/txline)"]
  BB["bracket_bond program"]
  TX["Txoracle program<br/>(TxLINE, daily Merkle roots)"]
  U -- buy / sell / redeem --> BB
  FE -- reads / signs --> BB
  REL -- "odds / scores / proof" --> BB
  BB -- "CPI validate_stat" --> TX
`} />

The user's money only ever moves in `bracket_bond`. Settlement only ever accepts
a proof that verifies against a root TxLINE published on Solana.

## How a round settles

1. **Buy** an outcome for a team at the current mark (implied probability from TxLINE odds).
2. **Watch** it mark up/down live as the real match moves; **exit** anytime.
3. **Each round**, a keeper fetches a stat proof from TxLINE and calls `settle_round`.
4. The program **CPIs into `Txoracle.validate_stat`** and requires the returned
   boolean to be `true` — an unproven result reverts. The losing outcome is
   eliminated; its collateral forfeits to the pot.
5. **Final** — the surviving outcome's holders redeem pro-rata, minus a protocol fee.
