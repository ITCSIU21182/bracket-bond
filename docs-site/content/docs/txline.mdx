TxLINE is a verifiable sports-data layer: real-time scores, events, and consensus
odds, each update cryptographically signed and anchored on Solana. Bracket Bond
consumes it as both the **pricing signal** and the **settlement source of truth**.

## Endpoints used

| Endpoint | Purpose in Bracket Bond |
| --- | --- |
| `POST /auth/guest/start` | guest JWT (`token`, 30-day) |
| `POST /api/token/activate` | activate the API token after the on-chain subscription |
| `/api/odds/stream` (SSE) | live StablePrice odds → the outcome **mark** |
| `/api/scores/stream` (SSE) | live scores / game state |
| `/api/scores/historical/{id}` | full score log (SSE) — used to find the settle `seq` |
| `/api/scores/stat-validation` | the **stat + Merkle proof** used to settle a round |
| `/api/fixtures/updates/{epochDay}/{hour}` | discover finished fixtures (`GameState === 3`) |
| on-chain `Txoracle.subscribe` / `validate_stat` | data authorization + trustless verification |

<Callout type="info">
  **TxL is data-only.** The TxL credit token authorizes data access; it is never
  used for staking or wagering. Bracket Bond escrows **native SOL** and settles
  payouts in SOL — a different asset, per the track rules.
</Callout>

## Free World Cup tier

Service level 1 (60-second delay, any network) is free — no token purchase, no
rate limits. Getting a working API token is a four-step flow, wrapped in
`scripts/txline/{subscribe,auth}.ts`:

```ts
const jwt     = await startGuestSession(host);                     // POST /auth/guest/start → { token }
const txSig   = await subscribeFreeTier(txoracle, wallet, conn,    // on-chain subscribe(1, 4 weeks)
                  { tokenMint });                                  //   Token-2022 ATA, service level 1
const apiToken = await activateApiToken({ host, jwt, txSig,        // POST /api/token/activate
                  wallet, leagues: [] });                          //   signs `${txSig}:${leagues}:${jwt}`
```

<Callout type="warn">
  `/api/token/activate` returns a **plain-text** token, not JSON — read it with
  `res.text()`, not `res.json()`. The dev host is intermittently flaky, so every
  call is wrapped in a small retry-with-backoff (`scripts/txline/net.ts`).
</Callout>

## The proof call

`/api/scores/stat-validation?fixtureId=&seq=&statKeys=1,2` returns a `summary`, the
`subTreeProof` / `mainTreeProof`, and per-stat `statsToProve` + `statProofs`. The
client maps those into `validateStatV2(payload, strategy)` against the
`daily_scores_roots` PDA (`epochDay` as a u16 LE seed):

```ts
const proof = await fetchStatValidation({ baseUrl: host, auth, fixtureId, seq, statKeys: [1, 2] });
const ix    = await buildValidateStatV2Ix(txoracle, proof, advancementStrategy(0, 1));
// relay through settle_round, or .view() it for a read-only boolean
```

See [Settlement](/docs/settlement) for how the program enforces the returned
boolean on-chain.

## What we hit

Genuine friction, surfaced by building against the live feed:

- **The documented stat-key period table doesn't match the live feed.** It inserts
  a half-time slot at `+2000`, shifting everything after it: `+5000` is actually
  **ET2** goals, and penalty-shootout goals are at **`+6000`** (6001/6002), not the
  documented `+5000`. A knockout on penalties therefore looks *level* at full time,
  so settlement needs a shootout predicate (`goals level` AND `pens differ`) or it
  resolves as a draw.
- **`period` is not a query param** — it's encoded in the stat-key offset; `&period=`
  is silently ignored. You *witness* a `seq` where the predicate holds.
- **Finality ≠ recency.** A later, non-final `seq` can arrive *after* the
  `game_finalised` event with the same score — settle on the `game_finalised` seq,
  never `max(seq)`.
- **`GameState` isn't a reliable "finished" flag** — the robust signals are the
  `game_finalised` event and `StatusId` (`10` = after-ET, `13` = on-penalties).
- `/api/token/activate` returns a **plain-text** token, not JSON (`res.json()` throws).
- The dev host times out often on ~1 MB payloads — wrap every call in retry-with-backoff.
- `/api/scores/historical` is SSE (`data:` lines), not a JSON array.
- The Txoracle IDL lives in the [`txodds/tx-on-chain`](https://github.com/txodds/tx-on-chain)
  examples — vendored here at `scripts/txline/idl/txoracle.json` (Apache-2.0).

## What we liked

The verifiability primitive is the whole product. Being able to `validate_stat`
a real match result on-chain — and get a clean boolean back — meant settlement
needed **no trusted oracle of our own**. That is exactly what a prediction market
has always been missing.
